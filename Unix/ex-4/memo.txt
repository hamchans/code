makefile は make でサーバ用のファイルが，make mydhcp_client でクライアント用のファイルが生成される．

サーバ側は，状態と送受信したメッセージの内容にしたがって状態遷移及びイベントを実行する．クライアントの IP アドレス使用期限の半分が経過したら SIGALRM を送信して REQUEST を待つ．さらに半分待って， REQUEST が来なければ IP アドレスを RELEASE する．また，クライアントから RELEASE を受信すると，IP アドレスを RELEASE する．複数クライアント対応は行っていない．また，タイムアウトは SIGALRM 送信時以外は実装していない．Unknown メッセージや予期していないメッセージを受信した場合は，イベントをエラーにして何も起きないようにしている．

クライアント側は，状態と送受信したメッセージの内容にしたがって状態遷移及びイベントを実行する．IP アドレス使用期限の半分が過ぎると SIGALRM を受信し，再度 REQUEST を自動的に送信する．別ターミナルから kill -HUP PID を行うと，RELEASE メッセージを送信し，終了する．
